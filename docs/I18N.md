# i18n strategy

## Locales and routes
- Required route prefixes from day one:
  - `/en` (English)
  - `/zh` (Chinese)
  - `/bn` (Bengali)

## App Router approach
- Use `app/[locale]/...` route segment.
- Validate locale in middleware and redirect unknown locales to default.
- Keep shared layout with locale dictionary injection.

## SEO notes
- SSR public pages for crawlability.
- Add `hreflang` alternate links across locale variants.
- Locale-aware sitemap entries.
- Canonical URLs per locale page.

## Content rollout
- Partial translation is acceptable initially.
- Missing keys should fallback to English while preserving locale route.
